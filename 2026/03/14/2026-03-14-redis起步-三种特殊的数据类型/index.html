# 三种特殊的数据类型

# HyperLogLogs（基数统计）

添加于Redis 2.8.9

# 基数

基数是指的是一个数据集中不重复元素的个数。比如说有数据集 [1, 2, 2, 3, 3, 3]。总元素有6个，基数有3个。

看pdai.tech的时候我就感觉它的概念怎么能用于实现统计。他说的其实不太对。基数因该是上面所说

# 优势

常用与各种统计计数，比如访问ip数之类的。但是为什么不使用Set呢？
事实上，HLL并不是准确的，它不会保存每个成员，它是通过观察哈希值中连续前导零的个数来估算基数。所以它的占用内存为12KB最多且误差率为0.81%

# Bitmap （位存储）

我也不知道什么时候加的

位图数据结构就是一个byte 数组或者 int/long 数组。里面就只存储0或1，它并不知道这一位意味着什么，所以就需要人为的赋予。
比如说一个用户今年的登录日期，这个key就可以是用户+年份，然后数组长度为365。后面就是偏移量的问题了。

# 优势

显而易见的优势在于节约了大量的存储空间

# 布隆过滤器

对了，布隆过滤器也是使用这个，
布隆过滤器的底层结构就是一个超级大的 Bitmap，加上几个不同的 Hash 函数。它常用于缓存穿透的防御。
它由两部分组成：

  1. 一个超级大的 Bitmap（全初始化为 0）。
  2. K 个不同的 Hash 函数（比如 3 个）。

录入数据（写操作）
当你要记录 "www.baidu.com" 时：
第 1 个 Hash 函数算出：10
第 2 个 Hash 函数算出：55
第 3 个 Hash 函数算出：99
布隆过滤器就会去 Bitmap 中，把第 10、55、99 位都涂黑（设为 1）。

# geospatial (地理位置)

Redis 的 Geo 在 Redis 3.2 版本就推出了! 这个功能可以推算地理位置的信息: 两地之间的距离, 方圆几里的人

两级无法直接添加，我们一般会下载城市数据(这个网址可以查询 GEO： http://www.jsons.cn/lngcode)！

有效的经度从-180度到180度。
有效的纬度从-85.05112878度到85.05112878度。
