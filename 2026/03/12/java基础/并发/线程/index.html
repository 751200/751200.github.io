# 线程的创建

  1. 继承 Thread 类（Java单继承机制，不推荐）。

    //就是继承，重写run，然后start()
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("线程执行");
    }
}

new MyThread().start();

  2. 实现 Runnable 接口（最常用，无返回值）。

    //Runnable就是Thread构造函数的一个参数
class MyRunnable implements Runnable {
    @Override
    public void run() {
        System.out.println("线程执行");
    }
}

new Thread(new MyRunnable()).start();

  3. 实现 Callable 接口配合 FutureTask，(有返回值，能抛出异常）。

    //这里相当于上面的run方法，返回一个3
Callable<Integer> task = () -> 1 + 2;

//futureTask是适配器，因为Thread构造要传Runnable
FutureTask<Integer> futureTask = new FutureTask<>(task);

new Thread(futureTask).start();

Integer result = futureTask.get();

  4. 通过线程池创建（真实项目中的唯一规范做法）。

    ExecutorService pool = Executors.newFixedThreadPool(5);

pool.submit(() -> {
    System.out.println("线程执行");
});

# 常见方法

# sleep()

  • 让线程休眠
  • 不释放锁
  • Thread类

# wait()

  • 让线程进入等待状态
  • 会释放锁
  • 会被notify()/notifyAll()唤醒
  • 必须在synchronized中

# wait()和sleep()的区别

对比点 wait() sleep()
所属类 Object 类的方法 Thread 类的方法
是否释放锁 会释放锁 不会释放锁
使用位置 必须在 synchronized 同步块/方法 中使用 可以在任何地方使用
唤醒方式 需要 notify() / notifyAll() 时间到了自动恢复
使用目的 线程间通信 让线程休眠一段时间

# join()

等待某个线程执行完

# notify()和notifyAll()

随机唤醒一个等待线程和唤醒所有等待线程

# 为什么必须在synchronized中

IllegalMonitorStateException异常

因为它们操作的是对象的 Monitor 等待队列，只有持有对象锁的线程才能调用
