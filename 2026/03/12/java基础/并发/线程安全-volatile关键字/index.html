# 核心语义

  1. 可见性
  2. 有序性（禁止指令重排）

volatile 不保证原子性

# volatile的可见性

线程修改变量
必须立即写回主内存

其他线程：

读取变量
必须从主内存读取

# JMM（java内存模型）的happens-before

volatile 实际上是通过 happens-before 规则工作的。

volatile 写
happens-before
volatile 读

为了实现这个 happens-before 关系，JVM 会在 volatile 操作周围插入 内存屏障（memory barrier）：

内存屏障一般有四种

  1. LoadLoad
  2. StoreStore
  3. LoadStore
  4. StoreLoad

意思：（屏障前面的什么操作）（屏障后面的什么操作）不能重排序，load-读，store-写

# 不保证原子性

多线程时可能这样：

线程A 读取 count=0
线程B 读取 count=0
线程A 写回 1
线程B 写回 1

最终结果：

count = 1

也就是说线程不安全

volatile 适合：

  • 状态标志
  • 单次写，多次读
  • 不依赖当前值的赋值
